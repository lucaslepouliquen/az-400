# AZ-400 - Structure et Domaines d'Examen

## 1. Prérequis

**OBLIGATOIRE d'avoir :**
- Azure Administrator Associate (AZ-104) **OU**
- Azure Developer Associate (AZ-204)

**Puis :**
- Passer AZ-400 = **Azure DevOps Engineer Expert**

---

## 2. Format de l'Examen

- **Durée :** 120 minutes
- **Questions :** 40-60 questions
- **Types :** QCM, scénarios, **labs pratiques** (Azure hands-on)
- **Score requis :** 700/1000
- **Aucune pénalité pour mauvaise réponse**

---

## 3. Les 5 Domaines (par ordre d'importance)

### 1. Design and Implement Build and Release Pipelines (40-45%) ⭐⭐⭐

**C'EST LE PLUS IMPORTANT - PRESQUE LA MOITIÉ DE L'EXAMEN**

Tu dois savoir :
- Automatiser builds avec Azure Pipelines
- Intégrer CI/CD end-to-end
- Setup staging environments avec approvals
- Implémenter déploiements multi-environnements (dev/test/staging/prod)
- Configurer continuous delivery
- Gérer artifacts
- Utiliser triggers
- Intégrations avec autres Azure services

### 2. Design and Implement Source Control (15-20%)

Tu dois savoir :
- Stratégies de branching (Git Flow, GitHub Flow, etc.)
- Pull requests et processus de review
- Intégration repos avec Azure Pipelines
- Policies : code reviews, build validation
- Permissions sur repos
- Git workflows

### 3. Configure Processes and Communications (10-15%)

Tu dois savoir :
- Setup environnement agile avec Azure Boards/Repos/Pipelines/Testing
- Établir canaux de communication (Teams, Slack, email alerts)
- Documenter processus (code reviews, testing signoff, release approvals)

### 4. Develop a Security and Compliance Plan (10-15%)

Tu dois savoir :
- Access controls via Azure AD et permissions
- Audit logging et monitoring sur pipelines
- Compliance infrastructure settings
- Adhérence aux régulations industry

### 5. Implement an Instrumentation Strategy (10-15%)

Tu dois savoir :
- Incorporer telemetry via Application Insights
- Monitor application performance
- Setup logging avec Log Analytics
- Alerting basé sur métriques critiques
- Dashboards et reporting analytics

---

## 4. Public Cible de l'Examen

**Qui devrait passer AZ-400 :**
- Developers construisant du software dans Azure
- Site Reliability Engineers (SRE) travaillant avec Azure
- Azure Administrators
- Toute personne voulant prouver son expertise DevOps sur Azure

**Profil idéal :**
- Expérience avec Azure infrastructure
- Connaissance DevOps practices
- Compréhension CI/CD
- Familiarité avec Agile/Scrum

---

## 5. Stratégies de Préparation

### Focus Areas (par priorité)

1. **Pipelines (40-45% de l'examen)**
   - Hands-on : créer des pipelines réelles
   - YAML syntax
   - Stages, jobs, steps
   - Triggers et conditions
   - Artifact management

2. **Source Control (15-20%)**
   - Git workflows pratiques
   - Branch policies
   - PR reviews

3. **Security & Compliance (10-15%)**
   - Azure AD integration
   - Secret management
   - Compliance scanning

4. **Instrumentation (10-15%)**
   - Application Insights setup
   - Log Analytics queries (KQL basics)
   - Alerting configuration

5. **Processes (10-15%)**
   - Azure Boards configuration
   - Agile workflows

### Approche d'Étude

**Ne pas faire :**
- Juste lire de la théorie
- Mémoriser sans comprendre
- Négliger les labs

**À faire :**
- **Hands-on practice** avec Azure DevOps (free tier disponible)
- Créer des projets réels avec pipelines
- Comprendre les **concepts** derrière les outils
- Pratiquer les labs (l'examen en contient)
- Utiliser Azure DevOps au quotidien si possible
