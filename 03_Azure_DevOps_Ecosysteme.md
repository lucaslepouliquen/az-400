# AZ-400 - Azure DevOps Écosystème

## 1. Définition

Service cloud Microsoft qui intègre seamlessly divers outils DevOps :
- Services Azure natifs
- Outils built-in
- Third-party tools

---

## 2. Core Features (à connaître absolument)

### 1. Azure Boards
- Project tracking
- Agile workflows
- Backlog management

### 2. Azure Repos
- Source control (Git)
- Version control

### 3. Azure Pipelines
- CI/CD automation
- Build et release pipelines
- **Plus gros domaine de l'examen (40-45%)**

### 4. Azure Test Plans
- Testing tools

### 5. Azure Artifacts
- Package management

---

## 3. Alignement avec Principes DevOps

- **CI/CD :** Pipelines automatisées
- **Collaboration :** Boards, intégrations Teams/Slack
- **Monitoring :** Application Insights, Log Analytics
- **Agile PM :** Sprints, backlogs, boards

---

## 4. Azure DevOps vs GitHub

### Azure DevOps
- Suite complète Microsoft
- Boards, Repos, Pipelines, Test Plans, Artifacts
- Intégration native Azure
- Entreprise-focused

### GitHub
- Appartient à Microsoft depuis 2018
- GitHub Actions = équivalent Azure Pipelines
- Peut s'intégrer avec Azure
- Plus orienté open-source/community

**Pour l'examen AZ-400 :**
- Focus principal sur **Azure DevOps**
- Mais connaître que GitHub peut aussi être utilisé
- GitHub Actions peut remplacer Azure Pipelines dans certains scénarios
